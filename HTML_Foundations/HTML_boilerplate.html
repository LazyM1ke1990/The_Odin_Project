HTML Boiler Plate

All HTML documents follow the same structure also known as 'boilerplate' that needs to
be in place before anything can be done. Firstly you would need to create a html
document using the '.html' extension. Most homepages would be called 'index' so your
first html document should be called 'index.html'

Every blank html page will start with a 'DOCTYPE' declaration. This is to help the
browser determine which HTML version is being used. The latest at present is HTML5,
The DOCTYPE for this version is '< !DOCTYPE html >'

Following the doctype declaration we need to provide the '< html >' element.
This element is known as the root element of the document, meaning all other elements
will be a decendant of it. This becomes important when we start using Javascript but
for now this just needs to be included in every html document.
Side Note... Majority of the time html elements have a 'lang' attribute along side it. this
is kind of like an 'Option'.

Example: < html lang="en" >< /html >

The lang attribute specifies which language the text content is in that element. So the html
document is going to be set to english according to the example above. 'en' is English.
providing this information allows assistive technologies to adapt accurately for instance
Screen readers.

After the html element, usually the 'Head' element comes next holding all the important
Meta-information about the webpage. The head element should not hold anything that
needs to be displayed on the webpage.

The Head element should be place inside of the HTML element and starts with the
'Meta' and 'Title' elements inside of itself usually. The meta element sets the charset
encoding of the webpage and is usually set to 'UTF-8' by default.
Example: < meta charset="UTF-8" >
Notice the Meta element is a self closing element so doesnt have closing tags!
Setting the encoding ensures the webpage will display special symbols in the browser.

The second part of the head element is the Title tags which is an element that produces
The webpages title in the browsers tabs. If we didnt have this then the tab would just
display the filename instead of a human readable title.
Example: < title >Homepage< /title >
Here the title of the webpage is 'Homepage' as opposed to 'index.html'
The more webpages open, the more you will umderstand the usefulness of titles.

There are more that can go into a head element but these two are the more important
at the moment. The only other thing that needs to be added to a boilerplate is the
'Body' element. This element goes in between the HTML tags but after the head tags
and it holds all the content which will be visable on the browsers webpage.

Top Tip

VSCode has a shortcut for a boilerplate template.
Selecting an '!' on an empty document and pressing tab to auto complete will provide a
template boilerplate already filled in and setup.
