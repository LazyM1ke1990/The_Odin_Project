Links and Images

Anchor Elements:

Anchor elements are used to create a link in html. They are difined by using '< a >' tags
with text or other html elements in between that becomes clickable. This clickable text
or html element will direct you to the links new page.

Example: < a >Homepage< /a >

The above example would set up a link to the 'Homepage' but this doesnt actually do
anything as an anchor tag doesnt know were its linking to on its own. We need to give
a destination for the link also. This is done by using a HTML attribute.
A HTML attribute is always placed in the opening tag of an element and is made up of
a name followed by a value even though a value isnt actually always needed with some
attributes. Attributes are used to give additional information to html elements.
The above element needs a 'href' attribute in the opening anchor tag with the value being
the destination we want the link to go to.

Example: < a href="www.homepage.com" >Homepage< /a >

In this example the anchor tag has the href attribute with the web address being the value
followed by 'Homepage' as the clickable text and finally finishing with closing tags.

Opening the links in a new tab:

The above example opens the link in the same tab your currently working in.
To change this all we need is another attribute to be placed in the opening anchor tag,
this attribute is the 'Target' attribute and is usually placed after the 'href' attribute.
The value of 'Target' by default is '_self' which means the link will open inside your
current working tab but if set to '_blank' then the link will open in a black tab/window.

If opening a link in a new tab or window, it is best practice to use the 'rel' attribute also
'rel' with a value of 'noopener' will stop any interactions between the new and old tabs.
This is ideal for security purposes because without it, the new page can use Javascript
to manipulate the original page. This causes a major security risk.

To be completely safe the attribute 'rel' should just have the value of 'noreferrer'.
'noreferrer' is the same as 'noopener' but adds the added benefit of blocking the
new page from knowing were the user came from originally as well as the interacting
with the previous page.

Absolute and Relative Links:

There are two kinds of links that we will generally make.

  • Links to pages on other websites via the internet
  • Links to pages located on own website

Absolute Links:

Absolute links are used to link pages from other websites on the internet.
This link will always contain a 'Scheme' and 'Domain' of the destination.
Scheme is like 'http://' and the domain is the 'Google.com' for example.

Relative Links:

Relative links are used to link to other pages on our own website. They do not include
a domain name since it is a page on the same site and assumes it will be the same.
Relative links only include the file path to the other page relative to the page the link
is created on.

Images:

To display an image in html we use the '< img >' element. The image element is a void
element so there is no closing tags as the element would be empty.
This is because we dont wrap the image content in tags, we embed an image to the page
using the 'SRC' attribute. The src attribute tells the browser were the image file is
located, just like the href attribute does. Images can be embedded using both absolute
and realtive paths.

Alt attribute:

Aswell as the src attribute to link the image to the page, you will need to provide an
'alt' attribute after the src attribute. The alt attribute helps describe the image to the
user if using say a screen reader or if the image fails to load. The description
needs to be as descriptive as possible.

Image Size Attribute:

Image size attributes are not required but help with layout. It is good practice to specify
the height and width dimentions of every image. These are usually altered with CSS
though.
