Elements and Tags

Almost all elements on a HTML page are just pieces of content wrapped in opening and closing HTML tags.
Opening tags tell the browser that it is at the start of a HTML element. They are defined by using angled brackets '<>'
An example would be the paragraph element THIS paragraph is enclosed in. It starts with an opening tag '< p >'
followed by the actual paragraph, then ending with a closing tag '< /p >' like the example below.

< p >Enter Paragraph here< /p >

The Closing tag is the opposite to the opening tag. The closing tag tells the browser that it has reached the end of that
element. The second difference is that it has a forward slash '/' immediately after the first angled bracket.
This is used to distinguish between opening and closing tags.

Elements are like containers for content. The opening and closing tags tell the browser what content the element holds. Using this information the browser can determine how to interpret and format the content held in the container tags.

HTML has a vast list of predifined tags for all kinds of different elements. It is important to use the correct tags
for content as it has a big impact on accessability for the users to the site.
Using the correct elements for content is called semantic HTML.

Some HTML tags do not have closing tags. These tags just have the opening tag like '< br >' or '< img >'
These are known as 'VOID' elements. This is because they are void of content, being void of content
means your unable to 'Wrapped' tags around anything like other tags do.
